如果多个逻辑控制流在时间上重叠执行，称为 **并发(concurrent)**，多核的并发可以实现并行。

并发不仅在内核，也在应用程序：
 - 访问慢速 I/O 设备时，交替执行 I/O 请求和其他有用的工作来利用并发；
 - 和计算机交互的人要求计算机有同时执行多个任务的能力；
 - 通过推迟部分工作以降低延迟（将耗时操作放在后台并发处理）；
 - 在多核机器上进行并行计算；

使用应用级并发的应用程序称为 **并发程序(concurrent program)**，现代操作系统提供了三种基本的构造并发程序的方法：
 - **进程**，每个逻辑控制流都是一个进程，由内核来调度和维护（进程拥有其独立的虚拟地址空间，需要使用显式的进程间通信机制 -- IPC(interprocess communication) 来通信）；
 - **I/O 多路复用**，应用程序在某一进程的上下文中调度逻辑流，一个逻辑流被分为多个阶段，根据当前状态决定下一步，并进行不同逻辑流之间的状态切换，它们共享同一地址空间；
 - **线程**，运行在单一进程上下文中的逻辑流，由内核进行调度，但共享同一进程的虚拟地址空间（进程 与 I/O 多路复用 的混合）；

本章将研究三种不同的并发编程技术，并以 11.4.9 节中的迭代 echo 服务器的并发版本为例。

# 12.1 基于进程的并发编程
构造并发程序最简单的方法就是进程，使用 `fork`、`exec` 和 `waitpid` 之类的函数。

而构造一个并发服务器的方法是：在父进程中接受客户端连接请求，然后创建一个新的子进程来为每个新客户端提供服务，父进程则继续监听新的连接。由于 `fork` 后父进程与子进程共享描述符，所以子进程需要关闭监听描述符，父进程关闭已连接描述符，以避免文件表项泄漏。

通过为每个客户端创建子进程，服务器可以并发地为多个客户端提供服务。

## 12.1.2 进程的优劣
父进程与子进程共享文件表，但是不共享用户地址空间：
 - 隔离性好，一个客户端崩溃不会影响其他客户端（优点）；
 - 独立的地址空间使得共享状态信息更加困难，需要使用显式 IPC 机制进程通信（缺点）；
 - `fork` 和进程切换的开销都比较大，且 IPC 的开销也很高（缺点）；

# 12.2 基于 I/O 多路复用的并发编程
由于服务器不仅需要响应网络客户端发起的连接请求，还有用户在键盘上键入。选择了一个就不能响应其他，故采用 **多路复用(I/O multiplexing)** 技术，即使用 `select` 函数，要求内核挂起进程，一次等待多个描述符，谁先准备好就处理谁 -- 将控制返回给应用程序。

`select` 有许多不同的使用场景，在此只讨论第一种场景：等待一组描述符 `fdset` 中某个 `fd` 可以读：
```c
int select(int n, fd_set *fdset, NULL, NULL, NULL);    // 返回已准备好的描述符的非零个数，若出错则为 -1

FD_ZERO(fd_set *fdset);           /* Clear all bits in fdset */
FD_CLR(int fd, fd_set *fdset);    /* Clear bit fd in fdset */
FD_SET(int fd, fd_set *fdset);    /* Turn on bit fd in fdset */
FD_ISSET(int fd, fd_set *fdset);  /* Is bit fd in fdset on? */
// 处理描述符集合的宏。
```
该函数处理类型为 `fd_set` 的集合，也成为 **描述符集合**：
 - 逻辑上是一个大小为 `n` 的位向量：$b_{n-1},…,b_1,b_0$，每个位 $b_k$ 对应一个描述符 $k$；
 - 当 $b_k=1$ 时，表示描述符 $k$ 在集合中；
 - 只可以对描述符集合做这三件事：
	 - 分配它们；
	 - 将一个此种类型的变量赋值给另一个变量；
	 - 用 FD_ZERO、FD_SET、FD_CLR 和 FD_ISSET 宏来修改和检查它们；

`select` 函数有两个输入：称为 **读集合** 的描述符集合(`fdset`) 和该读集合的基数(`n`)。

> `select` 函数会一直阻塞直到读集合中至少有一个描述符可以读了。

同时，`select` 会修改参数 `fdset` 指向的 `fd_set`，指向一个由读集合中准备好可以读的描述组成的一个集合，称为 **准备好集合(ready set)**。由于 `fd_set` 被修改了，为此需要在每次调用 `select` 时读更新读集合。

如图 12-6 展示了可以如何利用 `select` 来实现一个迭代 echo 服务器，可以接受标准输入上的用户命令：
![[NoteAboutStudy/attachments/Pasted image 20260326211405.png]]
 - 检查：`if (FD_ISSET(stdin))` -- 用户输入了命令，调用 `command()` 处理；
 - 检查：`if (FD_ISSET(listenfd))` -- 有客户端连接，调用 `accept、echo、close`；
一个进程可以同时处理多个 I/O，即 I/O 多路复用并发。

虽然使用了 `select` 函数进行阻塞，但仍然不是并发服务器，因为一旦连接到某个客户端就会连续回送输入行，直到客户端关闭连接。因此，如果键入一个命令在服务器和客户端之间结束之前将不会得到响应。为此，更细粒度的多路复用是服务器每次循环至多回送一个文本行。
