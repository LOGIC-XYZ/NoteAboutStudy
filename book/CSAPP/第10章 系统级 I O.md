**输入/输出(I/O)** 是主存和外部设备之间复制数据的过程，前者是 设备 -> 主存，后者是 主存 -> 设备。

所有语言运行时系统都提供执行 I/O 的较高级别的工具，例如 C 的 `printf` 和 `scanf` I/O 函数以及 C++ 的 重载操作符 `<<` 和 `>>`。在 Linux 系统中，是通过使用由内核提供的系统级 `Unix I/O` 函数来实现这些较高级别的 I/O 函数（底层实现）。

这一章将介绍 `Unix I/O` 和标准 `I/O` 的一般概念，并如何在 C 程序中使用。

---
# 10.1 Unix I/O
一个 **Linux 文件** 就是一个 $m$ 个字节的序列，并且将所有的 I/O 设备都抽象成 **文件**，所有的输入和输出都被当作对相应文件的读和写执行。

通过这种将设备都映射为文件的方式，使得 Linux 内核可以通过一个简洁且一致的应用接口，称为 `Unix I/O`，这样所有的输入和输出都能以一种统一的方式来执行：
 - **打开文件**，当程序想访问某个设备或文件时，必须先通过系统调用打开。内核会创建一个 ”打开文件表项“ 并返回一个小的非负整数 -- **文件描述符** 给程序。文件描述符指向所打开的文件，是进程级资源，而内核记录这个打开文件的所有信息；
 - **Linux shell 创建的每个进程开始时都默认打开三个文件**，
	 - **标准输入**，描述符为 0；
	 - **标准输出**，描述符为 1；
	 - **标准错误**，描述符为 2；
- **改变当前的文件位置**，对于每个打开的文件，内核都保持着一个 **文件位置 k**，初始为 0。这个文件位置是从文件开头起始的字节偏移量，是隐式维护的，读写操作会自动推进。应用程序能够通过执行 `seek` 操作，显式地设置文件的当前位置为 `k`；
- **读写文件**，
	- 一个读操作就是从文件复制 $n>0$ 个字节到内存，从当前文件位置 `k` 开始，然后增加到 $k+n$。如果 $k ≥ 文件大小 m$，再读就会触发 EOF(end-of-file)，应用程序能检测到这个条件（不过在文件结尾并没有 "EOF 符号"）；
	- 类似地，写操作就是从内存复制 $n>0$ 个字节到一个文件，从当前文件位置 `k` 开始，然后更新 $k$；
- **关闭文件**，释放打开文件表项 ，回收描述符到可用的描述符池中，并释放相关内核资源。如果程序异常终止，内核也会自动关闭所有打开文件，并释放它们的内存资源；

# 10.2 文件
每个 Linux 文件都有一个类型来表明在系统中的角色：
 - **普通文件(regular file)**，包含任意数据。其中应用程序需要区分 **文本文件** 和 **二进制文件**，文本文件只是应用层约定内容是只含有 ASCⅡ 或 Unicode 字符，并按 ”行“ 组织，每行以换行符 `\n` 结尾，二进制文件是其他所有文件。对内核而言，没有区别，都是字节流；
 - **目录(directory)**，包含一组 **链接(link)** 的文件，其中每个链接都将一个 **文件名** 映射到一个文件，这个文件也可能是另一个目录。目录至少包含两个条目：`.` 是到该目录自身的链接，`..` 是到其 parent directory 的链接( `cd ..` !)；
 - **套接字(socket)** 是用来与另一个进程进行跨网络通信的文件(见 11.4 节)；
其他类型的文件类型包括 **命名通道**、**符号链接** 以及 **字符和块设备**，不在本书的讨论范畴，但都遵循同一文件抽象。

Linux 内核将所有文件都组织成一个 **目录层次结构(directory hierarchy)**，由名为 `/` 的 **根目录** 确定。系统中的每个文件都可以通过从根目录出发的一条路径访问。如图 10-1 显式了 Linux 系统的目录层次结构的一部分：
![[NoteAboutStudy/attachments/Pasted image 20260223214740.png]]

作为其上下文的一部分，每个进程都有一个 **当前工作目录(current working directory)** 来确定其在目录层次结构中的当前位置，可以通过 `cd` 来跳转到指定的工作目录。

目录层次结构中的位置用 **路径名** 来指定。路径名是一个字符串，有两种形式：
 - **绝对路径(absolute pathname)**，以 `/` 开头，从根目录开始的路径。
 - **相对路径(relative pathname)**，以 `.` 或 `..` 开始，表示从指定的工作目录开始的路径；

# 10.3 打开和关闭文件
进程通过调用 `open` 函数来打开一个已存在的文件或创建一个新文件：
```c
int open(char *filename, int flags, mode_t mode);    // 若成功返回文件描述符，若出错为 -1
```
`open` 函数将 `filename` 转换为一个文件描述符，并返回该文件描述符。**描述符总是当前进程中尚未使用的最小描述符（是进程私有的索引）。**

`flag` 参数指明进程将如何访问这个文件：
 - O_RDONLY：只读；
 - O_WRONLY：只写；
 - O_RDWR：可读可写；
 也可以是一个或更多位掩码的或，作为行为修饰位：
  - O_CREAT：文件不存在时创建新文件；
  - O_TRUNC：文件已存在则清空已有内容；
  - O_APPEND：在每次写操作前，设置文件位置到文件结尾处（在结尾写入）；

`mode` 参数指定了新文件的访问权限位，这些位的符号名字如图 10-2 所示：
 ![[NoteAboutStudy/attachments/Pasted image 20260225212309.png]]
  - 只在 O_CREAT 创建新文件时生效。它指定文件的访问权限位（rwx），分别针对：
	- 文件拥有者（user）
	- 所属组（group）
	- 其他用户（other）
- 同时作为上下文的一部分，每个进程都有一个 `umask`（通过调用 `umask` 来设置），是进程的 “权限屏蔽位” ，决定创建文件时默认要去掉的权限；
- 所以，**实际权限 = mode & ~umask**

进程通过调用 `close` 函数关闭一个打开的文件：
```c
int close(int fd);    // 若成功返回 0，出错则为 -1（关闭一个已关闭的）
```

# 10.4 读和写文件
应用程序是通过分别调用 `read` 和 `write` 函数来执行输入和输出的。
```c
ssize_t read(int fd, void *buf, size_t n);    // 若成功返回读的字节数，若 EOF 则为 0，出错为 -1
ssize_t write(int fd, const void *buf, size_t n);    // 若成功返回写的字节数，出错为 -1
```
 - `read` 函数从描述符为 `fd` 当前的文件偏移处开始复制最多 `n` 个字节到内存位置 `buf`。返回值 -1 为出错，0 为 EOF，否则返回值表示的是实际读取的字节数量；
 - `write` 函数则从内存位置 `buf` 复制最多 `n` 个字节到描述符 `fd` 的文件当前偏移位置；

这两个系统调用都是 最多，而不是确保 -- 即传送的字节比应用程序要求的少，这些不足值(short count) 不表示错误，因为是出现这些情况的有：
 - 读到文件末尾了(EOF)；
 - 终端输入，当 fd 关联到终端时，内核通常按 “行” 返回数据。一次 `read` 可能只返回用户敲入的一行字符，而不是请求的全部字节；
 - 网络套接字或管道，底层缓冲区大小有限，数据分片到达，内核可能在任意时刻返回部分数据；

# 10.5 用 RIO 包健壮地读写
该节将讲述一个 I/O 包，称为 RIO(Robust I/O，健壮的 I/O 包)，可以自动处理 10.4 节中讲述的不足值。提供了两类不同的函数：
 - **无缓冲的输入输出函数**，在语义上等价于循环调用 `read` 或 `write`，直到达到指定字节数或遇到 EOF 或出现错误，不引入额外的用户级缓存，直接在内存和文件之间传输数据，适用于二进制数据的精确传输（只保证完整传输）；
 - **带缓冲的输入函数**，允许从文件读取较大块的数据，缓存到应用级缓冲区内，然后按需向调用者提供（类似 `printf` 的标准 I/O 函数提供的缓冲区，且是线程安全的，支持在同一个文件描述符上交错调用。同时这种机制减少系统调用次数，提高效率）；
---
## 10.5.1 RIO 的无缓冲的输入输出函数
通过调用 `rio_readn` 和 `rio_writen` 函数，应用程序可以在内存和文件之间直接传送数据。
```c
ssize_t rio_readn(int fd, void *usrbuf, size_t n);
ssize_t rio_writen(int fd, void *usebuf, size_t n);
// 若成功返回传送的字节数，若 EOF 返回 0（仅 rio_readn），若出错则为 -1
```
 - `rio_readn` 函数从描述符 `fd` 的当前文件位置最多传送 `n` 个字节到内存位置 `usrbuf`，`rio_writen` 函数从位置 `usrbuf` 传送 `n` 个字节到描述符 `fd`；
 - `rio_read` 函数在遇到 EOF 时允许返回不足值，`rio_writen` 函数决不会返回不足值；
 - 对同一个描述符，可以任意交错地调用 `rio_readn` 和 `rio_writen`；
如图 10-4 显示了 `rio_readn` 和 `rio_writen` 的代码。注意，如果 `rio_readn` 和 `rio_writen` 函数被一个从应用信号处理程序的返回中断，那么每个函数都会手动地重启 `read` 或 `write`。为了尽可能有较好的可移植性，允许被中断的系统调用，且在必要时重启它们。

## 10.5.2 RIO 的带缓冲的输入函数
目标有三个：
 1. 减少系统调用次数（提升效率）  
 2. 保持健壮性（处理 short count 和 EINTR）   
 3. 支持线程安全和混合读取模式
核心机制： 在用户态维护一个内部读缓冲区（rio_t 结构），当用户请求数据时：
- 若缓冲区有数据，直接从缓冲区拷贝；
- 若缓冲区为空，再调用 `read` 填充；

其中：
 - `rio_readlineb` 函数从文件 `rp` 读出下一个文本行（包括结尾的换行符），将其复制到内存位置 `usrbuf`，并且用 NULL 字符来结束这个文本行。最多读 `maxlen - 1` 个字节，余下的一个字符留给结尾的 NULL，超过的文本行被截断，并用 NULL 结束；
 - `rio_readnb` 函数从文件 `rp` 最多读 `n` 个字节到内存位置 `usrbuf`；
对同一描述符，两者的调用可以交叉进行（带缓冲的和无缓冲的函数不能交叉使用）。

RIO 读程序的核心是如图 10-7 所示的 `rio_read` 函数，是 Linux read 函数的带缓冲的版本：
![[NoteAboutStudy/attachments/Pasted image 20260226210525.png]]
当调用 `rio_read` 要求读 `n` 个字节时，读缓冲区内有 `rp->rio_cnt` 个未读字节：
- 如果缓冲区为空，就会通过调用 `read` 去填满；
- 如果调用 `read` 的返回值是一个不足值，则读缓冲区填充了一部分（不是错误，此时  `rp->rio_cnt` 是能读到的字节数）；
- 如果缓冲区非空，`rio_read` 就从读缓冲区复制 `min(n, rp->rio_cnt)` 个字节到用户缓冲区，并返回复制的字节数（选择前者是缓冲区数据足够，后者是数据不够）；

对于一个应用程序，`rio_read` 函数和 Linux read 函数有同样的语义：
 - 在出错时，它返回值 -1，并且适当地设置 `errno`；
 - 在 EOF 时，它返回值 0；
 - 如果要求的字节数超过了读缓冲区内未读的字节的数量，它会返回一个不足值。
 两个函数的相似性使得很容易通过 `rio_read` 代替 `read` 来创建不同类型的带缓冲的读函数。

# 10.7 读取目录内容
应用程序可以通过 `readdir` 系列函数来读取目录的内容。

```C
DIR *opendir(const char *name);    // 若成功返回处理的指针，出错返回 NULL
```
函数 `opendir` 以路径名为参数，返回指向 **目录流(directory stream)** 的指针，即目录项的列表。

```c
struct dirent *readdir(DIR *dirp);    // 若成功返回下一个目录项的指针，若没有下一个或出错返回 NULL
```
`readdir` 函数返回的是指向流 `drip` 中的下一个目录项的指针，如果没有则返回 NULL，如果出错返回 NULL 并设置 `error`（通过检查这个判断是流结束还是出错）。

每个目录项都是一个结构（至少包含这两个），其形式如下：
```c
struct dirent{
	ino_t d_ino;    // 文件位置（索引编号）
	char d_name;    // 文件名
}
```

函数 `closerdir` 关闭流并释放其所有资源：
```C
int closedir(DIR *drip);     // 成功返回 0，错误返回 -1
```

# 10.8 共享文件
可以用许多不同的方式来共享 Linux 文件，内核用三个相关的数据结构来表示打开的文件：
 - **描述符表(descriptor table)**，每个进程都有其独立的描述符表，它的表项是由进程打开的文件描述符来索引的，每个描述符表项指向 **文件表** 中的一个表项；
 - **文件表(file table)**，打开文件的集合是由一张 **文件表** 来表示的，所有进程共享这张表。每个文件表的表项组成包括当前的文件位置、**引用次数(reference count)** -- 即当前指向该表项的描述符表项数、以及一个指向 **`v-node` 表** 中对应表项的指针。（关闭一个描述符会减少相应的文件表表项中的引用计数，直到引用计数为零时，才会删除这个文件表表项）；
 - **v-node 表(v-node table)**，所有进程共享这张 v-node 表，每个表包含 `stat` 结构中的大多数信息，包括 `st_mode` 和 `st_size` 成员；

# 10.9 I/O 重定向
Linux shell 提供了 **I/O 重定向** 操作符，允许用户将磁盘文件和标准输入输出联系起来。

可以通过 `dup2` 函数来实现 I/O 重定向：
```C
int dup2(int oldfd, int newfd);    // 若成功返回非负的描述符，若出错则为 -1
```
`dup2` 函数复制描述符表表项 `oldfd` 到描述符表表项 `newfd`，覆盖描述符表表项 `newfd` 之前的内容。如果 `newfd` 已经打开了，`dup2` 会在复制 `oldfd` 之前关闭 `newfd`。最终目的即指向相同的文件表项。
