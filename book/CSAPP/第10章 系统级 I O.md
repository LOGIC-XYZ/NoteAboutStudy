**输入/输出(I/O)** 是主存和外部设备之间复制数据的过程，前者是 设备 -> 主存，后者是 主存 -> 设备。

所有语言运行时系统都提供执行 I/O 的较高级别的工具，例如 C 的 `printf` 和 `scanf` I/O 函数以及 C++ 的 重载操作符 `<<` 和 `>>`。在 Linux 系统中，是通过使用由内核提供的系统级 `Unix I/O` 函数来实现这些较高级别的 I/O 函数（底层实现）。

这一章将介绍 `Unix I/O` 和标准 `I/O` 的一般概念，并如何在 C 程序中使用。

---
# 10.1 Unix I/O
一个 **Linux 文件** 就是一个 $m$ 个字节的序列，并且将所有的 I/O 设备都抽象成 **文件**，所有的输入和输出都被当作对相应文件的读和写执行。

通过这种将设备都映射为文件的方式，使得 Linux 内核可以通过一个简洁且一致的应用接口，称为 `Unix I/O`，这样所有的输入和输出都能以一种统一的方式来执行：
 - **打开文件**，当程序想访问某个设备或文件时，必须先通过系统调用打开。内核会创建一个 ”打开文件表项“ 并返回一个小的非负整数 -- **文件描述符** 给程序。文件描述符指向所打开的文件，是进程级资源，而内核记录这个打开文件的所有信息；
 - **Linux shell 创建的每个进程开始时都默认打开三个文件**，
	 - **标准输入**，描述符为 0；
	 - **标准输出**，描述符为 1；
	 - **标准错误**，描述符为 2；
- **改变当前的文件位置**，对于每个打开的文件，内核都保持着一个 **文件位置 k**，初始为 0。这个文件位置是从文件开头起始的字节偏移量，是隐式维护的，读写操作会自动推进。应用程序能够通过执行 `seek` 操作，显式地设置文件的当前位置为 `k`；
- **读写文件**，
	- 一个读操作就是从文件复制 $n>0$ 个字节到内存，从当前文件位置 `k` 开始，然后增加到 $k+n$。如果 $k ≥ 文件大小 m$，再读就会触发 EOF(end-of-file)，应用程序能检测到这个条件（不过在文件结尾并没有 "EOF 符号"）；
	- 类似地，写操作就是从内存复制 $n>0$ 个字节到一个文件，从当前文件位置 `k` 开始，然后更新 $k$；
- **关闭文件**，释放打开文件表项 ，回收描述符到可用的描述符池中，并释放相关内核资源。如果程序异常终止，内核也会自动关闭所有打开文件，并释放它们的内存资源；

# 10.2 文件
每个 Linux 文件都有一个类型来表明在系统中的角色：
 - **普通文件(regular file)**，包含任意数据。其中应用程序需要区分 **文本文件** 和 **二进制文件**，文本文件只是应用层约定内容是只含有 ASCⅡ 或 Unicode 字符，并按 ”行“ 组织，每行以换行符 `\n` 结尾，二进制文件是其他所有文件。对内核而言，没有区别，都是字节流；
 - **目录(directory)**，包含一组 **链接(link)** 的文件，其中每个链接都将一个 **文件名** 映射到一个文件，这个文件也可能是另一个目录。目录至少包含两个条目：`.` 是到该目录自身的链接，`..` 是到其 parent directory 的链接( `cd ..` !)；
 - **套接字(socket)** 是用来与另一个进程进行跨网络通信的文件(见 11.4 节)；
其他类型的文件类型包括 **命名通道**、**符号链接** 以及 **字符和块设备**，不在本书的讨论范畴，但都遵循同一文件抽象。

Linux 内核将所有文件都组织成一个 **目录层次结构(directory hierarchy)**，由名为 `/` 的 **根目录** 确定。系统中的每个文件都可以通过从根目录出发的一条路径访问。如图 10-1 显式了 Linux 系统的目录层次结构的一部分：
![[NoteAboutStudy/attachments/Pasted image 20260223214740.png]]

作为其上下文的一部分，每个进程都有一个 **当前工作目录(current working directory)** 来确定其在目录层次结构中的当前位置，可以通过 `cd` 来跳转到指定的工作目录。

目录层次结构中的位置用 **路径名** 来指定。路径名是一个字符串，有两种形式：
 - **绝对路径(absolute pathname)**，以 `/` 开头，从根目录开始的路径。
 - **相对路径(relative pathname)**，以 `.` 或 `..` 开始，表示从指定的工作目录开始的路径；
